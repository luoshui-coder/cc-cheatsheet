# cc-cheatsheet

这是一个 **Claude Code 速查表** 静态网站项目。

项目定位：
- 以 `https://cc.storyfox.cz/` 为上游内容来源进行复刻。
- 在本地复刻基础上增加增强功能，而不是做完全一致的镜像。
- 当前仓库的主要目标，是在保留本地增强功能的前提下，持续同步上游网站的最新内容。

当前本地增强功能包括：
- 中英文切换
- 壁纸 / 横屏模式
- PNG 导出
- 响应式与打印优化
- 其他为本地使用场景补充的交互细节

项目结构：
- 整个网站目前以单个 [`index.html`](/Users/luoshui/ai_coding/pro/cc-cheatsheet/index.html) 为核心，CSS、HTML、JavaScript 和内容均内联其中。
- 无构建步骤，无运行时依赖。
- 本地预览时直接用浏览器打开 `index.html` 即可。

维护原则：
- 上游网站是“内容来源”。
- 本地仓库是“增强版复刻”。
- 同步更新时，优先更新内容，不要误删本地增强能力。

## 内容同步检查清单

当你需要把 `https://cc.storyfox.cz/` 的最新内容同步到本地这个增强版复刻项目时，使用这份清单。

规则：
- 只同步上游网站的内容变化。
- 不要移除本地增强功能，例如双语文案、壁纸模式、PNG 导出、响应式优化、自定义交互，除非上游内容发生了无法回避的结构性变化。
- 上游网站只作为“内容来源”，不是本地增强功能的覆盖来源。

建议按以下顺序检查：

1. 页头信息
- 版本号
- 最后更新时间
- 页面标题或副标题文案变化

2. 最近更新区块
- 用上游当前最新内容替换 changelog 列表
- 如果本地需要保留“新内容”提示，可为真正新增条目标记 `badge-new`

3. 键盘快捷键
- 常用控制
- 模式切换
- 输入前缀
- 会话选择器
- Transcript mode 或其他新增子区块

4. 斜杠命令
- 会话类命令
- 配置类命令
- 工具类命令
- 特殊命令
- 新增命令
- 已移除命令
- 命令说明文案变化

5. MCP 区块
- Scope 作用域
- 管理命令
- 各种限制、上限、`_meta` 覆盖项或其他新增约束

6. 记忆与文件
- `CLAUDE.md` 位置
- 规则 / 导入机制
- 自动记忆限制或相关文案更新

7. 工作流与技巧
- Effort / thinking 文案
- 权限被拒后的重试流程
- 上下文压缩行为
- 远程 / 调度相关说明

8. 配置与环境变量
- 配置文件
- 新增设置项
- 废弃或移除的设置项
- 环境变量列表
- 页脚环境变量摘要

9. 技能与代理
- Skill frontmatter 字段
- Agent frontmatter 字段
- 内置代理名称或说明变化

10. CLI 与命令标志
- 核心命令
- 新增标志或新配置流程
- 已移除标志或文案变更

11. 页脚摘要
- 权限模式文案
- 关键环境变量摘要

12. 本地增强功能兼容性检查
- `data-zh` / `data-en` 属性尽量保持为纯文本，确保现有语言切换脚本正常工作
- 除非本地脚本明确支持，否则不要在可翻译属性里嵌入 HTML
- 保持 `badge-new` 的现有行为不被破坏
- 不要改动壁纸模式 / 导出按钮相关逻辑，除非页面布局已经被上游内容挤坏

建议同步流程：

1. 打开上游网站，记录当前版本号和更新时间。
2. 按上面的区块顺序，对照本地 `index.html` 逐项检查。
3. 只做内容层更新，不动本地增强功能。
4. 搜索并清理过期内容：
   - 旧版本号
   - 旧日期
   - 本地仍保留的已移除命令
5. 做一遍本地增强功能冒烟检查：
   - 语言切换
   - 壁纸模式
   - 导出按钮
6. 最后检查 diff，确认除了必要内容同步外，没有误改本地增强逻辑。
